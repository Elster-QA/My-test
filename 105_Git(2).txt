***************КАММИТ С СЕРВЕРА НА ЛОК.МАШИНУ***************

1)Вносим изм. в файле(мой прим.FileOne.txt) на сервере Git(on line).
2)внизу стр. в поле "Discription" помечаем что мы изменили
3)наж. на "Commit changes" (изменения сохр. только на сервере Git-a,но НЕ на локалке)
4)в cmd (или в Git Bash(вызывается ПКМ))
  cat (FileOne.txt) показывает содержимое файла !!!ТОЛЬКО ДЛЯ GitBash!!!
5)в cmd (или в GitBash) пиш. git fetch (ком. снхрон.ветки 
(показывает что у нас есть изменения на Удал.Репоз.(сервер))
....можем проверить на сколько камитов у нас разница с сервером в cmd пиш. git status
6)в cmd пиш. git pull (наш фаqл на лок.маш. снхронизиров. с сервером Git-a)

***************ПРОСМОТР КАММИТОВ В CMD***************

1) в cmd  пиш. git log (инф. о всех коммитах и кто их делал и что входило)
2) в cmd пиш. q (виход из режима просмотра измен. )

...просмотр конкретного автора-изменений: git log --author имяавтора

...просмотр изменений в конкретном комите в cmd пиш. git show и хеш-комита
...просмотр изменений в ПОСЛЕДНЕМ комите в cmd пиш. git show
...просмотр изменений,кем и какие, были сделани в cmd пиш. git blame имяфайлаиразш
...просм. изм (кто сделал) в конкрт.сторе git blame имяфайлаиразш | grep номерстр !!!ТОЛЬКО ДЛЯ GitBash!!!
